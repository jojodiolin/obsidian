# 健康检查命令 — 知识库体检

你是这个 Obsidian vault 的 **维护者**。请扫描 `20-知识库/` 并找出问题。

## 检查范围
`$ARGUMENTS` 决定检查类型，默认 `all`：
- `links` — 双链完整性
- `orphans` — 孤儿笔记（无入链）
- `contradictions` — 数据矛盾
- `missing` — frontmatter 字段缺失
- `candidates` — 新文章候选（高频共现概念）
- `all` — 全部

## 各项检查细则

### links — 链接完整性
- 遍历所有 `[[xxx]]` 双链
- 找出**目标不存在**的链接（broken links）
- 找出**有占位 TODO 内容**的笔记（需补充）

### orphans — 孤儿笔记
- 找出**没有任何笔记链接到它**的 concepts
- 这些笔记"信息孤岛"，建议补充关联或归档

### contradictions — 数据矛盾
- 对同一概念/事实在不同笔记中的描述做对比
- 标记数值、定义、时间等不一致的地方

### missing — 字段缺失
- 检查每篇 concept 的 frontmatter
- 必填字段：`name`, `created`, `updated`, `sources`, `tags`
- 缺失则列出

### candidates — 新文章候选
- 统计概念之间的共现频率（在同一篇 source 中同时出现）
- 找出**高频共现但没有 topic 聚合文**的概念组合
- 建议 3-5 个 topic 主题

## 输出

写入 `40-元数据/体检报告/YYYY-MM-DD-HHMM.md`，结构：

```markdown
# 知识库体检报告 — YYYY-MM-DD HH:MM

## 概览
- 概念总数：N
- 主题总数：N
- 总字数：约 N 字
- 健康分：X/100

## 🔴 严重问题
...

## 🟡 改进建议
...

## 🟢 良好状态
...

## 📋 待办清单
- [ ] 修复 N 个 broken link
- [ ] 补全 N 篇笔记的 frontmatter
- [ ] 为 N 个孤儿笔记补充关联
- [ ] 撰写 topic：《xxx》
```

## 输出风格
- 末尾用一段简短中文总结体检结论
- 健康分计算：100 - (问题数 * 权重)，最低 0
- 不要自动修复，只报告（修复留给下次 `/编译` 或人工）
