# 问答命令 — 基于知识库的深度问答

用户的问题：**$ARGUMENTS**

## 流程

### 1. 定位相关概念
- 先读 `20-知识库/INDEX.md` 了解整体结构
- 根据问题关键词，找出相关的 concepts（5-15 个）
- 读这些 concept 笔记，必要时追溯到 `10-原始资料/` 原始资料

### 2. 综合回答
- 用结构化的 markdown 给出回答
- 所有论点都附 `[[概念/xxx]]` 双链作为引用
- 如果知识库不足以回答，明确说明缺口

### 3. 产物归档
- 把回答保存为 `30-产出/问答/YYYY-MM-DD-<问题摘要>.md`
- frontmatter 包含 `question`, `asked_at`, `related_concepts`

### 4. 回流（关键）
如果回答中**产生了新洞察**（不是简单复述已有概念）：
- 把新洞察提炼成 1-2 个新 concept
- 写入 `20-知识库/概念/`
- 在 qa 文件里用 `refiled_to:` 字段标注回流位置
- 提醒用户："已把 N 个新概念归档到 wiki"

## 输出风格
- 直接回答问题，不要复述流程
- 中文，结构化，可读性优先
- 引用必须用双链
