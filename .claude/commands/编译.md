# 编译命令 — 把原始资料"编译"进知识库

你是这个 Obsidian vault 的 **LLM 编译器**。请执行以下流程：

## 输入
- 目标范围：`$ARGUMENTS` 若为空，则扫描整个 `10-原始资料/`；否则只处理指定路径
- 知识库根目录：`20-知识库/`
- 索引文件：`20-知识库/INDEX.md`

## 流程

### 1. 扫描待编译资料
- 遍历 `10-原始资料/` 下所有 `.md` 文件
- 读取 frontmatter，识别 `compiled: false` 或无 frontmatter 的文件
- 若用户指定了路径，只处理该路径

### 2. 提取概念（核心步骤）
对每份原始资料：
- 识别其中的**核心概念、技术、人物、产品**
- 每个概念生成或更新一篇 `20-知识库/概念/<concept-name>.md`
- **更新而非覆盖**：若笔记已存在，在末尾追加 `## 更新 YYYY-MM-DD` 段落，注明新来源
- 新笔记采用以下模板：

```markdown
---
type: concept
name: <概念名>
aliases: []
created: <YYYY-MM-DD>
updated: <YYYY-MM-DD>
sources: []
tags: []
---

# <概念名>

## 一句话定义
<25 字以内>

## 详细说明
<主体内容，客观陈述>

## 关键事实
- 事实 1
- 事实 2

## 关联
- [[概念/相关概念A]]
- [[概念/相关概念B]]

## 来源
- [[10-原始资料/原始资料路径]]
```

### 3. 链接检查
- 确保所有 `[[概念/x]]` 双链目标存在
- 若目标不存在，先创建占位笔记（内容写 `TODO`）

### 4. 标记原始资料
更新原始资料的 frontmatter：
```yaml
compiled: true
compiled_at: <现在时间>
compiled_into:
  - 20-知识库/概念/xxx.md
  - 20-知识库/概念/yyy.md
```

### 5. 刷新索引
更新 `20-知识库/INDEX.md`，加入/更新本次新增的概念。

### 6. 输出日志
在 `40-元数据/编译日志/YYYY-MM-DD-HHMM.md` 写一份编译报告：
- 处理了几份原始资料
- 新增/更新了几个概念
- 列出新增概念的链接
- 发现的潜在问题

## 输出风格
- 末尾用一段简短中文总结本次编译成果
- 不要直接打印大量文件内容
