# 周度蒸馏 Prompt（给 Codex 自动化使用）

> **使用方式**：把下方 ``` 包裹的整段 prompt 复制到 Codex 的定时任务配置里，设为每周五 18:00 触发。
> Codex 会作为 agent 在 vault 根目录运行，自动产出本周蒸馏报告。

---

## 配置要点（先读）

- **工作目录**：`C:\Users\Administrator\Desktop\ObsidianVault`
- **执行频率**：每周五 18:00（避开周一 8:00 Horizon 抓取 + 9:00 /体检 提醒）
- **输出位置**：`每周蒸馏/YYYY-WW-周蒸馏.md`（如 `2026-W25-周蒸馏.md`）
- **不修改**：除输出文件外，**禁止修改 vault 内任何其他文件**（只读分析）
- **失败兜底**：若任何步骤报错，在报告末尾的"执行日志"段记录错误并继续，不要中途退出

---

## Prompt 正文（复制以下内容到 Codex）

```
你是这个 Obsidian vault 的「周度蒸馏师」。今天是周五，请执行本周的知识库蒸馏。

## 工作目录
`C:\Users\Administrator\Desktop\ObsidianVault\`（以下所有路径都相对此目录）

## Vault 结构速览
- `10-原始资料/` 原始资料（每日速递 / 网页剪藏 / 小红书 / 论文 / 图片）
- `20-知识库/` LLM 编译产物（INDEX.md + 概念/ 主题/ 人物/ 摘要/）
- `30-产出/` 问答 / 幻灯片 / 图表产出
- `40-元数据/` 编译日志 / 体检报告
- `每周蒸馏/` ← 你这次的输出目录
- `00-收件箱/` 待处理新笔记

## 本周时间窗口
本周一 00:00 ~ 现在（用文件 mtime 判断）

## 执行步骤

### 1. 数据盘点
- 扫描 `10-原始资料/` 下本周新增/修改的 `.md` 文件，列出文件名 + 字数
- 扫描 `20-知识库/概念/` `20-知识库/人物/` `20-知识库/主题/` 下本周新增/修改的笔记
- 统计：本周新增原始资料数、新增概念数、累计概念总数

### 2. 主题聚类（核心）
对本周新增的概念做语义聚类，找出 **2-4 个本周主题**。每个主题：
- 主题名（10 字以内）
- 涉及概念清单
- 一段 50-100 字的本主题综述
- 代表性原始资料链接

如果概念太少（<5 个）无法聚类，直接跳到第 3 步并在报告里说明"本周样本不足，仅做信息归档"。

### 3. 内容选题（你是 AI 博主的助理）
基于本周主题 + 概念，**建议 3-5 个适合 AI 自媒体写作的选题**。每个选题：
- 标题（带流量感的钩子标题，15-25 字）
- 角度（一句话说明这篇文章的差异化视角）
- 目标读者（技术人 / 投资人 / 普通爱好者，三选一或多个）
- 引用素材（来自本周哪些概念/原文）
- 时效性（"必须本周发" / "本月内" / "长期可写"）

### 4. 健康检查（轻量版）
- 列出 `20-知识库/INDEX.md` 里出现但实际文件不存在的 broken link
- 列出 0 入链的孤儿笔记（前 10 个）
- 列出 frontmatter 缺 `tags` 或 `sources` 字段的概念（前 10 个）
- 不要修复，只报告

### 5. 缺口识别
本周的原始资料里，哪些信息点只是被"提了一句"但缺乏深度展开？列 3-5 个值得后续 `/编译` 深挖的候选项。

### 6. 下周展望
基于本周主题的趋势推断，给 2-3 条"下周应该关注什么"的建议（Horizon 配置可能需要新增的 RSS 源、值得主动搜索的关键词等）。

## 输出文件

**文件名**：`每周蒸馏/YYYY-WW-周蒸馏.md`
- YYYY = 当前年份
- WW = 当前 ISO 周数（两位数，如 W25）
- 例：`每周蒸馏/2026-W25-周蒸馏.md`
- 若该文件已存在，**不要覆盖**，改名为 `YYYY-WW-周蒸馏-v2.md`

**frontmatter**：
```yaml
---
type: weekly-distillation
created: <当前日期 YYYY-MM-DD>
week: <YYYY-WW>
covers:
  raw_count: <本周新增原始资料数>
  concept_count: <本周新增概念数>
  total_concepts: <累计概念总数>
  topics: [<主题1>, <主题2>, ...]
tags: [weekly, distillation]
---
```

**正文结构**（用一级 / 二级标题组织）：
1. `# 本周蒸馏 · YYYY-WW`（开头放一段 100-150 字的本周综述，给读者一个 TL;DR）
2. `## 📊 数据盘点`（表格：原始资料数 / 新增概念 / 累计概念）
3. `## 🎯 本周主题`（每个主题一个三级标题）
4. `## ✍️ 内容选题`（表格或卡片式）
5. `## 🩺 健康检查`（broken link / 孤儿 / 缺字段）
6. `## 🔍 缺口与深挖候选`
7. `## 🔮 下周展望`
8. `## 📝 执行日志`（你这次跑了几步、耗时、遇到的错误）

## 输出风格要求
- **中文为主**，专业术语保留英文（如 LLM, RLHF, MoE）
- 不要罗列原文，**做信息压缩**（一篇 1000 字日报压缩到 50 字摘要）
- 选题标题要有 **AI 博主风格**（不是论文标题），可以带数字、对比、争议
- 引用素材用 Obsidian 双链：`[[概念/xxx]]` `[[10-原始资料/每日速递/AI-Daily-xxx]]`
- 末尾不需要"总结"段落，执行日志之后直接结束

## 重要约束
- **只读分析**，不写任何文件除了最终输出文件
- 不要修改 `INDEX.md`、概念文件、原始资料
- 不要执行 `/编译` 或 `/体检`（那是别的循环做的事）
- 所有结论都要有 vault 内的具体文件作为依据，不要编造
- 遇到不确定性，明说"基于本周样本有限，建议观察"
```

## Codex 定时配置示例

如果 Codex CLI 支持类 cron 的定时（参考 `codex schedule --help`）：

```bash
# 每周五 18:00 执行（避开 :00 整点）
codex schedule add \
  --name "weekly-distillation" \
  --cron "37 18 * * 5" \
  --cwd "C:\Users\Administrator\Desktop\ObsidianVault" \
  --prompt-file "每周蒸馏/prompt-for-codex.md"
```

> cron 用 `37 18 * * 5`（18:37）而非 `0 18 * * 5` 是为了避开整点请求高峰。
> 若 Codex 不支持 cron 表达式，用对应的"每周五 18:37"自然语言配置。

## 与其他循环的关系

| 循环 | 频率 | 动作 | 主导者 |
|------|------|------|--------|
| Ingest | 每天 8:00 | Horizon 抓 AI 新闻 | Windows Task Scheduler |
| Compile | 不定期 | `/编译` 抽概念入 wiki | 你（Obsidian 里手动） |
| Lint | 每周一 9:00 | `/体检 all` | Windows 弹窗提醒 + 你手动触发 |
| **Distill** | **每周五 18:00** | **本周蒸馏 + 选题** | **Codex 自动**（本 prompt） |

四循环互补：抓取 → 编译 → 体检 → 蒸馏，覆盖一周完整生命周期。
